前端需要了解的数据结构

Word Count:   1.4k

复杂度分析

时间复杂度

  • 主要求代码循环多少次
  • 最好时间复杂度
  • 最坏时间复杂度

空间复杂度

数组

特性

  • 线性表
  • 内存空间连续
  • 成员类型相同

根据下标查找的时间复杂度为 O(1)

高效的随机访问

低效的插入和删除操作

  • 插入

    • 为了把第 k 个位置腾出来，给新来的数据，我们需要将第 k ～ n 这部分的元素都顺序地往后挪一位

  • 删除类似

  • 时间复杂度 O(n)

Java ArrayList

  • 支持动态扩容

    • 耗费性能
    • 建议  预先指定数组大小

链表

特征

  • 通过指针将松散的内存块连在一起

  • 链表节点除了存储数据，还要存储下个节点的地址

  • 插入和删除数据的速度很快

    • O(1)

  •  随机访问很慢

    • O(n)

单链表

双向链表

  • 每个  节点不止有一个后续指针 ，还有一个前驱指针指向前面的节点
  • 占用更多的空间
  • 插入和删除  操作更高效

循环链表

  • 比单链表具有环形结构

二分查找

O(logn)

必须是有序的数组

递归

时间复杂度

  • O(n)

二叉树

树的种类

  • 满二叉树

  • 完全二叉树

    • 叶子节点都在最底下两层，最后一层的叶子节点都靠左排列
    • 便于存储于数组

二叉树存储

  • 链式存储

    • 每个节点有三个字段
    • 一个存储数据，另外两个存储左右子节点指针
    • 图例

  • 数组存储

    • 我们把根节点存储在下标 i = 1 的位置，那左子节点存储在下标 2 _ i = 2 的位置，右子节点存储在 2 _ i + 1 = 3 的位置

    • 图例

    • 如果是非完全二叉树，会浪费更多的存储空间

    • 如果某棵二叉树是一棵完全二叉树，那用数组存储无疑是最节省内存的一种方式

      • 也是为什么完全二叉树要求最后一层的子节点都靠左的原因

    • 数组顺序存储的方式比较适合完全二叉树，其他类型的二叉树用数组存储会比较浪费存储空间

二叉树遍历

  • 深度优先遍历

    • 图例

    • 前序遍历

      • 先打印当前，然后先左后右

    • 中序遍历

      • 先左，然后当前，后右

    • 后序遍历

      • 先左后右，最后当前节点

    • 具体实现

      • 递归法

        • 使用递归将 tree(object)的节点 push 到数组当中

      • 非递归法

        • 使用栈这个数据结构

  • 广度优先遍历

    • 按照层去遍历

    • 步骤

      • 使用队列来实现
      • 1 初始化一个队列，把根节点如队列
      • 当队列为非空时，循环执行步骤 3 到 4，否则执行结束；
      • 出队列取得一个结点，访问该结点；
      • 若该结点的左子树为非空，则将该结点的左子树入队列，若该结点的右子树为非空，则将该结点的右子树入队列；

二叉查找树

  • 支持动态数据集合的快速插入，删除，查找操作

  • 树的任意一个节点，其左子树中的每个节点的值，要小于这个节点的值，右节点的值要大于这个节点的值

  • 查找

    • 如果要查找的数据比根节点的值小，那就在左子树中递归查找；如果要查找的数据比根节点的值大，那就在右子树中递归查找

  • 插入

    • 如果要插入的数据比节点的数据大，并且节点的右子树为空，就将新数据直接插到右子节点的位置；如果不为空，就再递归遍历右子树，查找插入位置。同理，如果要插入的数据比节点数值小，并且节点的左子树为空，就将新数据插入到左子节点的位置；如果不为空，就再递归遍历左子树，查找插入位置。

    • 值相同

      • 把相同的值放在同一个节点上
      • 把值插入到右树

  • 删除

    • 删除节点没有子节点

      • 置位 null

    • 删除节点只有一个子节点

      • 更新父节点，删除子节点

    • 删除节点有两个节点

      • 找到节点的右树中最小的节点，替换到要删除的节点

    • 还有一个方法

      • 单纯将要删除的节点标记为 已删除，但是并不真正从树中将节点删除

  • 效率

    • 在二叉查找树中，查找、插入、删除等很多操作的时间复杂度都跟树的高度成正比。两个极端情况的时间复杂度分别是 O(n) 和 O(logn)，分别对应二叉树退化成链表的情况和完全二叉树。

搜索算法

广度优先搜索

  • 地毯式层层推进，从顶点开始，一次往外遍历

  • 需要借助队列来实现

  • 时间复杂度 O（E）

    • E 表示边的个数

  • 空间复杂度 O（V）

    • V 表示顶点个数

深度优先搜索

  • 使用回溯思想，使用递归来实现

    • 借助栈来实现

  • 时间复杂度 O（E）

  • 空间复杂度 O（V）

排序算法

冒泡排序

  • O(n2)

插入排序

  • O(N2)

快排

  • O(nlogn)

  • 步骤

    • 随机选取数组中一个数 p
    • 其他数比 p 大的一组，比 p 小的一组
    • 递归调用以上步骤，最后合并

  • 循环实现和递归实现

归并排序

  • O(nlogn)

  • 分而治之的思想

  • 两步走

    • 将数组分成两部分，并分别排序
    • 将排序好的部分合并

桶排序

  • O(n)
